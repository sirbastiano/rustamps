StaMPS-style processing in native Rust

The production binary implements the available single-master workflow in process. It preserves the established dataset and MAT-artifact contracts without loading a Python runtime or invoking external scientific programs.

Stage map

StagePurposeCompletion artifact
1Load candidate phase, geometry, baselines, and metadataPATCH_*/ps1.mat
2Estimate coherence and topographic-error termsPATCH_*/pm1.mat
3Select persistent-scatterer candidatesPATCH_*/select1.mat
4Weed weak, adjacent, and duplicate candidatesPATCH_*/weed1.mat

Scientific invariants

  • MAT serialization preserves MATLAB dimensions, complex values, sparse/logical/character values, typed empties, NaN, and Inf.
  • One-based persisted identifiers are converted to zero-based indices only inside Rust kernels.
  • Stage 6 keeps master and dropped-interferogram columns zero.
  • Stage completion artifacts are committed only after the full output bundle succeeds.
  • Missing sensor metadata and unsupported algorithm branches fail instead of receiving plausible fabricated defaults.

Stage 6 correctness and restartability

The in-process Rust solver minimizes the integer-flow objective to convergence. It writes fingerprinted per-interferogram checkpoints below .pystamps-stage6/. Phase, geometry, baseline, date, selection, or solver changes invalidate affected checkpoints, and phuw2.mat appears only when all solves complete.

A positive flow-pass bound is rejected because that approximation failed scientific validation.

Explicit speed/accuracy trade

The supported speed lever is grid spacing. The balanced profile multiplies spacing by 4 and the fast profile by 10, reducing dense grid cells by roughly 16 and 100 times respectively. Both profiles retain a converged flow solve.

pystamps --config configs/stage6-fast.yaml run \
  --dataset DATASET --start-step 6 --end-step 8

Grid coarsening changes spatial sampling. Validate the chosen profile against a strict-grid result before accepting it for a dataset.

Verification is the acceptance gate

pystamps verify --run RUN --golden GOLDEN
pystamps --config configs/stage6-fast.yaml verify \
  --run RUN --golden GOLDEN \
  --profile scientific --through-stage 6

Strict mode checks exact structure and configured numeric tolerances. Scientific mode permits only bounded outliers with hard caps. Unwrapped phase is never wrapped merely to pass comparison.

Compatibility boundary

Small-baseline processing, external unwrapping, non-native kernel providers, reference replay, and unsupported Stage 7/8 alternatives fail closed. See the native runtime contract for the complete current list.

The historical implementation under oracle/ is used only for developer comparisons through explicit make oracle-* targets. It is not a production backend.
