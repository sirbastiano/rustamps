What every pipeline step does

Rustamps implements the supported single-master persistent-scatterer workflow as eight native stages. Stages 1–4 work independently inside each patch, Stage 5 finishes each patch and joins them, and Stages 6–8 work on the merged dataset.

A small glossary

  • Acquisition: one radar image captured on one date.
  • Reference or master: the one acquisition to which every secondary image is aligned.
  • Interferogram: the phase difference between the reference and one secondary acquisition.
  • Persistent scatterer (PS): a pixel whose radar response stays stable enough to measure phase through time.
  • Wrapped phase: phase stored inside a repeating −π to +π interval. Unwrapping restores the missing whole 2π cycles.
  • Perpendicular baseline: the cross-track separation between two satellite positions. Residual elevation error changes phase in proportion to this baseline.
  • Patch: a manageable, slightly overlapping piece of the scene processed before the global merge.

Before Stage 1 — prepare candidates

Purpose: turn a SNAP StaMPS export into the raw point files that the scientific stages consume.

  1. Read every co-registered RSLC and normalize its amplitude so dates are comparable.
  2. Compute amplitude dispersion at each pixel. A stable amplitude gives the first, deliberately broad PS candidate set.
  3. Reject pixels with near-zero signal or invalid longitude, latitude, or height.
  4. Split the scene into overlapping patches and extract each candidate’s wrapped phase across all interferograms.
  5. Copy sensor heading and wavelength into parms.mat.

Writes: PATCH_*/pscands.1.*, patch.list, and parms.mat. See the Inputs guide for the SNAP chain and exact export tree.

Stage 1 — organize the observations

Question answered: what was observed at every candidate point, on which dates, and at what position?

  1. Read candidate row/column, wrapped complex phase, longitude/latitude, height, and amplitude dispersion.
  2. Read dates, the single reference date, per-interferogram baselines, and sensor geometry from the SNAP parameter and baseline files.
  3. Put acquisitions in chronological order and create the reference column in the expected position.
  4. Convert geographic coordinates to a local metric XY system, remove unusable rows, and spatially sort the candidates.
  5. Reindex every related array together so point IDs, phase, height, and baselines still refer to the same rows.

Writes: ps1.mat (point/date geometry), ph1.mat (complex phase), bp1.mat (per-point baselines), and optional amplitude, height, and look-angle files. ps1.mat is the completion marker.

Why it matters: every later fit assumes these arrays are aligned. Stage 1 makes that contract explicit once.

Stage 2 — estimate phase stability

Question answered: which candidates have phase that can be explained consistently, rather than mostly by noise?

  1. Remove the reference phase so every interferogram uses the same phase origin.
  2. Place nearby points on a spatial grid and apply the CLAP phase filter to estimate the slowly varying neighborhood signal.
  3. For each candidate, fit K, the phase term proportional to perpendicular baseline. It represents residual topographic or look-angle error.
  4. Fit a constant phase term C, calculate the remaining phase residual, and turn residual consistency into temporal coherence.
  5. Repeat the estimate with P-square random-noise or SNR weighting until convergence or the configured iteration limit.

Writes: pm1.mat, containing coherence, K/C, filtered patch phase, residuals, and the random-coherence reference. Higher coherence means the candidate behaves more like a stable scatterer.

Stage 3 — select persistent scatterers

Question answered: where should the coherence cutoff be, and which candidates clear it?

  1. Group candidates by amplitude dispersion so points with similar amplitude behavior are compared fairly.
  2. Compare the observed coherence distribution with the distribution expected from random phase.
  3. Choose a threshold that limits the expected random points by density or percentage, then keep candidates strictly above it.
  4. Normally refilter and refit the selected subset, recalculate the threshold, and reject unstable topographic-error estimates.

Writes: select1.mat, which records the source indices, threshold model, and final keep mask. This is statistical selection, not the final spatial cleanup.

Stage 4 — weed unreliable or redundant points

Question answered: which selected points remain trustworthy when their location and temporal noise are considered?

  1. Apply the Stage 3 keep mask and, when requested, keep only the best member of a group of adjacent candidates.
  2. Remove duplicate locations and optionally zero-height points.
  3. Correct the selected phase with the estimated topographic term.
  4. Connect nearby points with a Delaunay network and measure phase noise along those edges through time.
  5. Reject points whose temporal standard deviation or maximum noise exceeds the configured limits.

Writes: weed1.mat, with the final per-patch keep masks and noise measurements. The topology is used to judge local consistency; it does not change geographic coordinates.

Stage 5 — promote and merge the patches

Question answered: how do the independently cleaned patches become one scene without duplicated overlap points?

  1. Apply all selection and weeding masks to phase, geometry, baseline, height, and model arrays.
  2. Correct and rereference phase, then write each patch’s version-2 products such as ps2.mat, ph2.mat, and pm2.mat.
  3. Use each patch’s non-overlap bounds to assign one owner to shared borders.
  4. Resolve remaining duplicate coordinates, join the owned rows, recompute local XY, and sort the merged scene.
  5. Estimate an interferogram-level residual standard deviation for later noise weighting.

Writes: version-2 products in every patch and at the dataset root. Root ifgstd2.mat is the completion marker.

Scope boundary: after Stage 5, Stages 6–8 read the merged root products rather than looping over patches.

Stage 6 — unwrap the phase

Question answered: how many whole 2π cycles are missing from the wrapped phase at each point and date?

  1. Load merged phase and geometry, exclude the reference and any deliberately dropped interferograms, and apply available phase corrections.
  2. Coalesce nearby PS on an unwrap grid. Optionally apply the Goldstein prefilter to reduce high-frequency noise.
  3. Build spatial neighbor edges and estimate space-time phase differences and their uncertainty.
  4. Translate those uncertainties into integer-flow costs, then solve each interferogram to convergence for the most consistent cycle corrections.
  5. Interpolate the solved grid phase back to the original PS, restore required corrections, and keep reference/dropped columns at zero.

Writes: reusable uw_*.mat intermediates and the final phuw2.mat. The final file appears only after every requested interferogram succeeds.

Restart behavior: fingerprinted per-interferogram checkpoints live under .pystamps-stage6/. Changes to phase, geometry, dates, baselines, selections, or scientific solver settings prevent stale reuse.

Stage 7 — estimate spatially correlated look-angle error

Question answered: how much of the unwrapped phase still follows baseline because the elevation/look angle is slightly wrong?

  1. Optionally remove a degree-1 spatial ramp and center phase on the configured geographic reference area.
  2. Fit unwrapped phase against each point’s perpendicular-baseline history with the native L2 method.
  3. Use acquisition timing and Stage 5 interferogram noise in the fit.
  4. Compare each estimate with its Delaunay neighbors and clamp extreme K/C values to a spatially plausible envelope.
  5. Build the phase correction predicted by the smoothed look-angle-error field.

Writes: scla2.mat with per-PS estimates and phase correction, plus scla_smooth2.mat for the smoothed field.

Stage 8 — estimate space-time correlated noise

Question answered: what broad phase pattern is shared by nearby points and varies through time?

  1. Start from unwrapped phase and subtract the Stage 7 SCLA, constant, and optional ramp terms.
  2. Optionally fit and remove a spatial plane from selected interferograms.
  3. Apply a temporal Gaussian high-pass so slow temporal behavior is separated from shorter changes.
  4. Apply a spatial Gaussian low-pass to retain the part correlated over nearby points.
  5. Reference the estimate consistently and force the reference-acquisition noise column to zero.

Writes: scn2.mat: ph_scn_slave (estimated correlated noise), ph_hpt (temporally high-passed phase), and ph_ramp (optional planes).

Interpretation: scn2.mat is a correction/noise product, not by itself a line-of-sight velocity map.

Run, resume, and rerun safely

# Preview writes without changing the dataset
rustamps run --dataset DATASET --start-step 1 --end-step 8 --dry-run

# Run a specific contiguous range
rustamps run --dataset DATASET --start-step 3 --end-step 6

# Resume: skip stages whose completion markers already exist
rustamps run --dataset DATASET --start-step 0 --end-step 8

A positive start stage means “rerun from here.” Before a stage writes, Rustamps removes products that depend on the old result. Rerunning a patch stage also invalidates merged products; rerunning a merged stage removes only later merged products. Every output bundle is transactional, so a marker is not published for half a stage.

Speed, verification, and supported scope

The reviewed Stage 6 speed lever is grid spacing. Scale 4 makes each grid dimension about four times coarser; scale 10 makes it about ten times coarser. Both still solve integer flow to convergence, but coarsening changes spatial sampling and must be validated against the strict grid.

rustamps --config configs/stage6-fast.yaml run \
  --dataset DATASET --start-step 6 --end-step 8

rustamps --config configs/stage6-fast.yaml verify \
  --run DATASET --golden STRICT_RESULT \
  --profile scientific --final-products-only --through-stage 6

The validated end-to-end contract is single-master PS processing. Small-baseline mode, external unwrapping, reference replay, non-native providers, atmospheric subtraction in Stage 7, and unsupported Stage 8 kriging fail before output publication. See Configuration for supported parameters and tolerance rules.
