From Sentinel-1 SLC to Rustamps inputs

Rustamps does not read a raw Sentinel-1 .SAFE product directly. First use SNAP to align a single-master stack, form differential interferograms, and run StaMPS Export. Then rustamps prep snap converts that Gamma-format export into patch inputs for Stage 1.

Use Level-1 SLC, not GRD. SLC pixels retain the complex phase needed for interferometry. Terrain-corrected GeoTIFFs, unwrapped rasters, generic SNAP .dim products, and GRD amplitude images are not direct Rustamps inputs.

1. Choose and download the acquisitions

Search the Copernicus Browser for Sentinel-1 Level-1 IW SLC products. Keep one internally consistent stack:

  • the same relative orbit and pass direction;
  • the same acquisition mode, polarization, and overlapping sub-swath/bursts;
  • one reference date used for every reference–secondary interferogram;
  • at least four secondary acquisitions, because the current SNAP StaMPS Export operator requires that minimum.

The four-secondary rule is only an exporter floor, not a claim that five dates are scientifically sufficient. Use enough acquisitions for the stability, temporal coverage, and deformation signal your study needs.

2. Build the stack in SNAP

The exact menu labels can move between SNAP versions, but the processing logic stays the same.

  1. TOPS Split: select the same sub-swath, polarization, and common bursts from every SLC. This removes data that cannot participate in the shared radar grid.
  2. Apply Orbit File: replace predicted orbit metadata with precise or restituted orbit vectors when available. Better orbit geometry improves co-registration.
  3. Choose one reference: use the InSAR Stack Overview or a reviewed project choice, then keep that date fixed for the whole stack.
  4. Back Geocoding: resample every secondary SLC onto the reference radar grid using orbit data and a DEM. Inspect the result for alignment problems.
  5. Enhanced Spectral Diversity: refine azimuth alignment when the retained area spans multiple TOPS bursts.
  6. Interferogram Formation: combine each reference–secondary pair into complex wrapped phase. Remove flat-Earth and topographic phase, and request elevation plus orthorectified latitude/longitude bands.
  7. TOPS Deburst: join adjacent bursts for both the co-registered SLC stack and interferogram stack. If you subset, do it after deburst and use identical bounds on both products.

Do not terrain-correct, Goldstein-filter, SNAPHU-unwrap, or generically multilook the products before export. Rustamps expects radar-grid complex samples and performs its own later filtering and unwrapping.

3. Run StaMPS Export

Open Radar → Interferometric → PSI / SBAS → StaMPS Export and provide, in this order:

  1. the debursted, co-registered SLC stack containing the reference and secondaries;
  2. the matching debursted differential-interferogram stack with elevation and orthorectified latitude/longitude.

Select the PSI format and one output directory. The current operator writes Gamma-format rasters into rslc, diff0, geo, and sometimes dem. Older pair-based SNAP graphs may create separate subfolders; merge like-named folders into one dataset root before continuing.

SNAP documentation next suggests mt_prep_gamma. For Rustamps, stop before that command and use rustamps prep snap instead.

4. Check the exported tree

INSAR_20200101/
├── rslc/
│   ├── 20200101.rslc
│   ├── 20200101.rslc.par
│   └── 20200113.rslc
├── diff0/
│   ├── 20200101_20200113.diff
│   └── 20200101_20200113.base
├── geo/
│   ├── 20200101.lon
│   ├── 20200101.lat
│   └── elevation_dem.rdc
└── dem/                         # optional SNAP output

Repeat the dated RSLC and reference–secondary .diff/.base files for every acquisition. Rustamps requires a single reference date across all baselines and interferograms. The reference .rslc.par supplies raster geometry, heading, radar frequency, and orbit-derived values.

SNAP’s Gamma writer produces big-endian 32-bit values: interleaved real/imaginary floats for .rslc and .diff, and scalar floats for longitude, latitude, and height. Do not byte-swap or convert these files.

5. Prepare Rustamps patches

rustamps prep snap \
  --dataset /data/INSAR_20200101 \
  --master-date 20200101

The command reads all RSLC amplitudes, computes amplitude dispersion, keeps pixels with finite geometry below the threshold, divides them into patches, and extracts each candidate’s wrapped phase history. It publishes PATCH_*, patch.list, and sensor values in parms.mat.

OptionDefaultMeaning
--master-dateDetectReference date. It can be inferred from INSAR_YYYYMMDD or one unambiguous diff0 prefix.
--amp-dispersion0.4Keep candidates strictly below this amplitude-dispersion threshold. Lower is more selective.
--range-patches1Number of patch columns.
--azimuth-patches1Number of patch rows.
--range-overlap50Extra range pixels included at each internal patch edge.
--azimuth-overlap50Extra azimuth pixels included at each internal patch edge.
--forceOffReplace existing prepared PATCH_* inputs.
 
rustamps status --dataset /data/INSAR_20200101
rustamps run --dataset /data/INSAR_20200101 --start-step 1 --end-step 1

Running Stage 1 is the complete metadata preflight: it also parses every baseline file and the full reference geometry. A failed stage does not publish a partial completion artifact.

Common export failures

  • No master can be inferred: pass --master-date YYYYMMDD.
  • Missing .lon, .lat, or elevation: enable orthorectified coordinates and elevation before StaMPS Export.
  • Unexpected raster size: one product was subset or debursted differently, or the Gamma parameter dimensions do not match the binaries.
  • No candidates pass: inspect the RSLC stack and geocoding rasters before relaxing --amp-dispersion.
  • Several reference dates appear: rebuild as one single-master PSI stack; Rustamps does not silently reinterpret an SBAS network.
