Run flow

Inspect, rehearse, then run.

This page is the shortest correct path through pySTAMPS. It is intentionally practical: every step is something you can run on a dataset copy without needing to understand every scientific detail first.

Step 1: inspect the dataset

Start by asking pySTAMPS what it sees in the dataset root.

pystamps status --dataset inputs_and_outputs/InSAR_dataset_test

This command does not process the data. It reports the merged stage and patch stages it can infer from the artifacts already present.

Step 2: dry-run the full pipeline

A dry-run is a safe rehearsal. It shows what pySTAMPS would do without executing the stages.

pystamps run --dataset inputs_and_outputs/InSAR_dataset_test --start-step 1 --end-step 8 --dry-run

Step 3: run on a copy, not on your only original

pySTAMPS writes artifacts into the dataset tree. Make a run copy first.

cp -a inputs_and_outputs/InSAR_dataset_test /tmp/InSAR_dataset_test_run
pystamps run --dataset /tmp/InSAR_dataset_test_run --start-step 1 --end-step 8
For debugging, shorten the range. For example, --start-step 6 --end-step 8 is useful when you only need to study late merged stages.

Step 4: validate outputs

Verification is now documented in Verification. It covers direct CLI compare commands, repository parity audit scripts, and stale-copy safety rules.

Need worker tuning?

pystamps run --dataset DATASET_DIR --start-step 1 --end-step 8 --io-workers 12 --cpu-workers 4

Need a config file?

runtime:
  backend: auto
  io_workers: 8
  cpu_workers: 0
  stage7_chunk_ps: 100000
  stage8_chunk_edges: 200000
  enable_mat_stage_cache: true
