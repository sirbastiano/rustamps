Troubleshooting

Questions, answered natively

Start with dataset status and the dry-run plan; both are read-only.

Why does a run report missing input artifacts?

Inspect discovery first with rustamps status --dataset DATASET and rustamps describe-inputs --stage all --dataset DATASET. For a compatible but unprepared SNAP export, run rustamps prep snap --dataset DATASET.

How do I resume completed stages?

Use --start-step 0. Resume mode reports skipped_existing only for valid completion artifacts. An explicit start step in 1..8 is a rerun and invalidates dependent later products before processing.

What happens after an interrupted Stage 6?

Rustamps reuses only matching, checksummed per-interferogram checkpoints. It publishes the final phuw2.mat transactionally after every solve completes.

Can I enable CuPy, a GPU provider, or an external solver?

No. The production configuration is native Rust only. External or GPU kernel providers and SNAPHU-style solver delegation are rejected before output is written; installing Python packages does not change the compiled runtime.

Does MAT v7.3 require a system HDF5 library?

No. Rustamps handles supported MAT v5 and v7.3 data through its pure-Rust I/O boundary. Unsupported MAT structures return an explicit error instead of falling back to another runtime.

How should I investigate a verification failure?

Keep the source and golden trees intact, rerun on a writable copy, and review the complete verifier report. Use --through-stage N only for an intentionally stage-scoped run. See the verification guide.

Supported science boundary: the validated production path is the single-master persistent-scatterer workflow. Unsupported small-baseline and nonstandard Stage 6 modes fail closed.
