Getting started with native Rustamps

Rustamps is one Rust binary that prepares compatible SNAP data, runs the eight-stage StaMPS-style workflow, and verifies its artifacts. Production processing has no external scientific executable or language runtime.

Install and inspect

cargo build --release --locked
cargo install --path . --locked
rustamps describe-backends
rustamps describe-inputs --stage all

Prepare data safely

Process a copy, because a positive stage range replaces selected artifacts and invalidates dependent later products.

cp -a /path/to/source_dataset /path/to/run_dataset
rustamps status --dataset /path/to/run_dataset

If the copy is a compatible SNAP StaMPS export without patch inputs:

rustamps prep snap --dataset /path/to/run_dataset

Preparation derives wavelength and heading from the master RSLC metadata. Missing or invalid sensor metadata fails explicitly.

Run or resume

rustamps run --dataset /path/to/run_dataset \
  --start-step 1 --end-step 8 --dry-run
rustamps run --dataset /path/to/run_dataset \
  --start-step 1 --end-step 8

Use --start-step 0 to resume from existing completion artifacts. Stage 6 also reuses only atomic per-interferogram checkpoints whose scientific fingerprint still matches.

Choose the grid deliberately

The default native configuration preserves the dataset unwrap grid and solves integer flow to convergence. The balanced and fast profiles reduce dense grid cells by increasing spacing; this can produce large speed gains at lower spatial resolution.

rustamps --config configs/stage6-balanced.yaml run \
  --dataset /path/to/run_dataset --start-step 6 --end-step 8

A bounded flow-pass approximation is not supported because it failed scientific validation.

Verify before accepting results

rustamps verify \
  --run /path/to/run_dataset \
  --golden /path/to/reference_dataset

The verifier checks artifact structure as well as numeric values. Read the full science guide before adopting a speed profile.

Historical reference implementation

The retained Python tree is an isolated development oracle. It is neither installed with nor called by the native command. Developers reproduce reference results only through explicit make oracle-* targets.
