Install the native Rust binary

The production package is the root Cargo package. Rust 1.89 or newer is the only language toolchain required.

Conda package

The rustamps 0.3.0 development-channel Conda package contains only the native Rust CLI. It does not install Python, a system HDF5 library, SNAPHU, or another external scientific executable.

Install and smoke-test it in a clean environment:

conda create -n rustamps -c sirbastiano/label/dev -c conda-forge rustamps=0.3.0
conda run -n rustamps rustamps --version
conda run -n rustamps rustamps describe-backends

Compiled packages target linux-64, linux-aarch64, osx-64, osx-arm64, win-64, and win-arm64. Linux packages require glibc 2.17 or newer, macOS packages require macOS 11 or newer, and the CLI is neither a musl nor noarch package.

Windows ARM64 remains experimental on the dev label until native ARM64 CI passes. Promotion of the exact tested artifacts to main requires an explicit release decision.

Build from a checkout

git clone https://github.com/sirbastiano/rustamps.git
cd rustamps
cargo build --release --locked
cargo install --path . --locked
rustamps --help
rustamps describe-backends

describe-backends should report the native provider and an empty runtime-external-dependency list.

Supported systems

Native source installation is gated on 64-bit little-endian Linux (GNU and musl), macOS, and Windows, for both x86_64 and ARM64. CI builds, tests, installs, and launches every supported target with Rust 1.89.

32-bit, big-endian, BSD, mobile, WebAssembly, and unusual network-filesystem transaction semantics are not currently supported.

Production dependency boundary

The installed command does not import Python or execute MATLAB, SNAPHU, Triangle, or another scientific program. It also does not require a system HDF5 library.

Build with --locked so the checked dependency graph is used. The root Cargo.lock is the release dependency contract.

Confirm the build

cargo fmt --all -- --check
cargo test --workspace --locked
cargo build --release --locked
cargo tree -p rustamps

See the native release process for the complete gate.

Developer oracle

The historical implementation is isolated under oracle/ for reference reproduction. Its development environment is not installed with, loaded by, or used as a fallback by the production binary.

make oracle-setup
make oracle-test
