Native configuration contract

Configuration is loaded from YAML or JSON by the Rust binary. Unknown fields and unsupported values are errors.

Production runtime fields

FieldDefaultConstraint
runtime.backendnativenative; auto is an alias
runtime.stage2_kernel_backendnativenative; auto is an alias
runtime.stage6_solvernativein-process native solver only
runtime.cpu_workers0non-negative; zero uses available CPUs
runtime.stage6_grid_scale1.0finite and greater than zero
runtime.stage6_max_flow_passes0must be zero for convergence

Verification fields

tolerance supports profile, relative/absolute tolerance, wrapped and exact key sets, maximum outlier fraction, hard maximum absolute error, and per-key policies. A nonzero outlier allowance requires scientific mode and a hard cap.

Removed contracts

There is no external-tools block, reference-replay mode, or per-kernel provider map in the supported runtime. Legacy no-op fields are accepted only at their defaults and otherwise rejected so they cannot silently mislead users.
