Make the science safer

Contribute to Rustamps

Keep changes focused, preserve the native production boundary, and pair scientific behavior changes with reproducible evidence.

Set up a checkout

git clone https://github.com/sirbastiano/rustamps.git
cd rustamps
cargo build --release --locked
cargo test --workspace --locked

Before opening a pull request

cargo fmt --all -- --check
cargo test --workspace --locked
cargo build --release --locked
cargo tree -p rustamps

Include the exact commands and fixtures used to verify stage math. Keep the source or golden dataset intact and run on a writable copy.

Preserve contracts

Respect MATLAB-style indexing, column-major I/O boundaries, checkpoint fingerprints, and fail-closed mode checks.

Stay native

The production dependency graph must not gain a Python, MATLAB, SNAPHU, Triangle, or system-HDF5 runtime.

Show evidence

Use deterministic fixtures and the scientific verifier when changing kernels, filters, solvers, or tolerances.

The retained Python tree and oracle/pyproject.toml are reference tooling only. Use explicit make oracle-* targets when a change needs oracle comparison.

Read the full repository guidance in CONTRIBUTING.md and start a discussion in GitHub Issues.
