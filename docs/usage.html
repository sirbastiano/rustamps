Native command workflows

Inspect inputs and state

pystamps describe-backends
pystamps describe-inputs --stage all
pystamps status --dataset DATASET

These commands are read-only. describe-backends reports the native runtime boundary.

Plan, run, and resume

pystamps run --dataset DATASET --start-step 1 --end-step 8 --dry-run
pystamps run --dataset DATASET --start-step 1 --end-step 8
pystamps run --dataset DATASET --start-step 0 --end-step 8
  • A positive start step recomputes the range and invalidates dependent later artifacts.
  • Start step 0 resumes from valid completion artifacts.
  • --cpu-workers N bounds the native Rayon pool; zero uses the available CPU budget.
  • Stage 6 resumes matching per-interferogram checkpoints and publishes phuw2.mat only when complete.

Run a controlled stage range

pystamps run --dataset DATASET --start-step 3 --end-step 5
pystamps run --dataset DATASET --start-step 6 --end-step 8

Each stage validates required upstream artifacts. Unsupported compatibility branches fail before publishing substitute results.

Apply an explicit native profile

pystamps --config configs/native-kernels.yaml run \
  --dataset DATASET --start-step 1 --end-step 8

pystamps --config configs/stage6-fast.yaml run \
  --dataset DATASET --start-step 6 --end-step 8

The fast profile coarsens Stage 6 grid spacing but retains converged flow optimization. Compare it scientifically with a strict-grid run before accepting it.

Verify a run

pystamps verify --run RUN --golden GOLDEN
pystamps verify --run RUN --golden GOLDEN --through-stage 6
pystamps --config CONFIG verify \
  --run RUN --golden GOLDEN --profile scientific

Strict is the default. Scientific mode still enforces structure and configured hard maximum errors.

No runtime fallback

All production stages execute in native Rust. Python/CUDA backends, external phase unwrapping, reference replay, and per-kernel backend overrides are rejected rather than silently changing the computation.
