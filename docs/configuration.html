Native runtime configuration

Pass YAML or JSON with rustamps --config CONFIG COMMAND. Unknown fields and unsupported legacy values fail during loading.

Minimal explicit configuration

runtime:
  backend: native
  stage2_kernel_backend: native
  stage6_solver: native
  cpu_workers: 0
  stage6_grid_scale: 1.0
  stage6_max_flow_passes: 0

Active runtime fields

FieldContract
backendnative; legacy auto normalizes to native
stage2_kernel_backendnative or native alias auto
stage6_solverIn-process native solver; native, auto, and backend normalize to native
cpu_workersZero uses the available CPU budget; positive values bound the worker pool
stage6_grid_scaleFinite positive multiplier for Stage 6 grid spacing
stage6_max_flow_passesMust be zero so the flow optimizer converges

Verification tolerances

tolerance:
  profile: scientific
  rtol: 1.0e-5
  atol: 1.0e-7
  max_outlier_fraction: 0.001
  max_abs: 0.05
  key_tolerances:
    ph_uw:
      atol: 0.001
      max_outlier_fraction: 0.001
      max_abs: 0.05

Nonzero outlier fractions require the scientific profile and a finite hard max_abs cap. Exact-key and wrapped-key policies remain enforced.

Rejected legacy options

External solvers, Python/CUDA providers, reference replay, per-kernel overrides, bounded Stage 6 flow passes, and non-default inert worker, chunk, cache, checkpoint, or debug fields are rejected. This prevents a configuration from appearing effective while being ignored.

Checked profiles

  • configs/native-kernels.yaml: strict grid scale 1.0.
  • configs/stage6-balanced.yaml: coarser scale 4.0.
  • configs/stage6-fast.yaml: coarser scale 10.0.

Balanced and fast profiles trade spatial resolution for speed; both keep flow optimization converged and require scientific verification.
