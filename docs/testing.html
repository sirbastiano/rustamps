Testing

Running tests

uv run pytest -q
uv run pytest -q tests/test_dataset.py

The repository includes committed unit tests. Dataset-backed parity tests are kept in-tree as optional checks and skip cleanly when the local validation datasets are absent.

Test structure

  • Default unit tests cover config loading, dataset discovery, verification classification, acceleration helpers, and Stage-7 helper math.
  • Dataset-backed parity tests and audits require local inputs under inputs_and_outputs/ and are not part of the minimal fresh-clone contract.
  • Parity checks are orchestrated through scripts and CLI workflows rather than through package import APIs alone.

Coverage

There is no tracked CI workflow or Makefile in this standalone repo snapshot. Validation is driven directly through uv run pytest, python -m build, twine check, and the parity audit scripts.

In-code assertions and explicit exceptions define many behavior guarantees where test coverage is expected to assert failure conditions.

Validation workflow

Stage verification contract and audit metadata are represented by pystamps.parity_contract constants and exported helpers.

The parity audit requires local validation datasets. When those datasets are not present, use the default unit test slice instead of the audit commands below.

python - <<'PY'
from pystamps.parity_contract import build_parity_contract
print(build_parity_contract('inputs_and_outputs'))
PY
uv run python scripts/validate_audit.py \
  --datasets inputs_and_outputs/InSAR_dataset_test \
  --output inputs_and_outputs/validation_runs/latest_audit.json
