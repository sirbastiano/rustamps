Testing

Running tests

make test
uv run pytest -q

The project does not include a large test suite in-tree in this snapshot; command entries from docs are retained for maintainers.

Test structure

  • Unit-level checks are expected around config loading and dataset discovery.
  • Functional checks are typically stage-contract driven via verification artifacts.
  • Parity checks are orchestrated through scripts and are not part of this API surface.

Coverage

Coverage is inferred via CI quality gates and project make audit workflows; direct line-coverage is not exposed by this module set.

In-code assertions and explicit exceptions define many behavior guarantees where test coverage is expected to assert failure conditions.

Validation workflow

Stage verification contract and audit metadata are represented by pystamps.parity_contract constants and exported helpers.

python - <<'PY'
from pystamps.parity_contract import build_parity_contract
print(build_parity_contract('inputs_and_outputs'))
PY
