## Local rules for docs work
- Read code from adjacent source paths and write only inside `docs/`, `examples/`, changelog, or architecture-note paths.
- Keep examples runnable and version-aligned with the code.
- Run docs lint/build or link checks before handoff.
- Update migration notes, configuration docs, and architecture notes when the task changes behavior or design.
- Do not change production source code from a docs-only task.
